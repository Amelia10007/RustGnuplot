Variants

  • Auto - Lets the value scale automatically

Enum PlotOption

An enumeration of plot options you can supply to plotting commands, governing things like line width, color and others

Variants

  • PointSymbol(char) - Sets the symbol used for points. The characters are as follows:
    • . - dot
    • + - plus
    • x - cross
    • * - star
    • s - empty square
    • S - filled square
    • o - empty circle
    • O - filled circle
    • t - empty triangle
    • T - filled triangle
    • d - empty del (upside down triangle)
    • D - filled del (upside down triangle)
    • r - empty rhombus
    • R - filled rhombus

  • Caption(&'self str) - Sets the caption of the plot element. Set to empty to hide it from the legend.

  • LineWidth(float) - Sets the width of lines.

  • Color(&'self str) - Sets the color of the plot element. The passed string can be a color name (e.g. "black" works), or an HTML color specifier (e.g. "#FFFFFF" is white). This specifies the fill color of a filled plot.

  • LineType(DashType) - Sets the dash type. Note that not all gnuplot terminals support dashed lines. See DashType for the available types.

  • FillAlpha(float) - Sets the transparency of a filled plot. 0.0 - fully transparent, 1.0 - fully opaque

  • FillRegion(FillRegion) - Sets the fill region. See See FillRegion for the available regions.

Method fill_between

fn fill_between<'l, Tx: DataType, X: Iterator<Tx>, Tyl: DataType,
                YL: Iterator<Tyl>, Tyh: DataType,
                YH: Iterator<Tyh>>(&'l mut self, x: X, y_lo: YL, y_hi: YH,
                                   options: &[PlotOption]) -> &'l mut Axes2D

Plot a 2D scatter-plot of two curves (bound by y_lo and y_hi) with a filled region between them. FillRegion plot option can be used to control what happens when the curves intersect. If set to Above, then the y_lo < y_hi region is filled. If set to Below, then the y_lo > y_hi region is filled. Otherwise both regions are filled.

Arguments

  • x - Iterator for the x values
  • y_lo - Iterator for the bottom y values
  • y_hi - Iterator for the top y values
  • options - Array of PlotOption controlling the appearance of the plot element

Implementation for Figure<'self> where <'self>

A figure that may contain multiple axes

Example

let x = [0, 1, 2];
let y = [3, 4, 5];
let mut fg = Figure::new();
{
fg.axes2d()
.lines(x.iter(), y.iter(), [Caption("A line"), Color("black")]);
}
fg.show();

Method set_terminal

fn set_terminal<'l>(&'l mut self, terminal: &'self str,
                    output_file: &'self str) -> &'l mut Figure<'self>

Sets the terminal for gnuplot to use, as well as the file to output the figure to. Terminals that spawn a GUI don't need an output file, so pass an empty string for those.

There are a quite a number of terminals, here are some commonly used ones: * wxt - Interactive GUI * pdfcairo - Saves the figure as a PDF file * epscairo - Saves the figure as a EPS file * pngcairo - Saves the figure as a PNG file
